Lumen API 文档

自托管 AI 网关与对话管理平台 — 支持 9 家内置厂商与自定义 OpenAI 兼容端点

JWT 认证

所有 API 通过 Bearer Token 鉴权，支持过期与刷新

多用户隔离

用户数据完全隔离，管理员可配置细粒度权限

多厂商支持

9 家内置厂商 + 自定义 OpenAI 兼容端点，30+ 模型

基础信息

Base URL {your_domain}/api
认证方式 Authorization: Bearer <token>
Content-Type application/json
默认管理员 admin / admin123 首次登录需改密

快速开始

安装与启动

# 克隆项目
git clone https://github.com/your-org/lumen.git
cd lumen

# 安装依赖
pip install -r requirements.txt

# 启动服务（默认端口 5000）
python app.py

第一个 API 请求

1. 登录获取 Token

curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

2. 发送对话

curl -X POST http://localhost:5000/api/chat/send \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_token>" \
  -d '{"message":"你好，请介绍一下你自己"}'

认证接口

所有认证接口均不需要 Token。除注册外，其余接口始终可用（不受接口开关影响）。

POST /api/auth/register

注册新用户。需管理员开启开放注册开关，否则返回 403。

请求体
{
  "username": "string",    必填
  "password": "string",    必填 最少 6 位
  "email": "string",       可选
  "daily_limit": 100       可选 null=继承全局
}
成功响应 200
{
  "success": true,
  "message": "注册成功",
  "token": "eyJ...",
  "user": { "id": 1, ... }
}
POST /api/auth/login

用户登录，返回 JWT Token（有效期 24 小时）。

请求体
{
  "username": "admin",
  "password": "admin123"
}
成功响应 200
{
  "success": true,
  "token": "eyJ...",
  "user": { ... },
  "must_change_password": true
}
GET /api/auth/me 需认证

获取当前登录用户的详细信息。

Headers
Authorization: Bearer <token>
POST /api/auth/change-password 需认证

修改当前用户密码。

请求体
{
  "old_password": "current_password",
  "new_password": "new_strong_password"
}
POST /api/auth/logout 需认证

登出当前用户，使 Token 失效。

对话接口

核心 AI 对话功能，支持多轮上下文、流式输出、多模型切换。所有接口均需认证，受每日限额与权限控制。

POST /api/chat/send 需认证

发送消息并获取 AI 回复。自动加载对话历史构建多轮上下文。

请求体
{
  "message": "你好",           必填
  "conversation_id": 1,        可选 继续已有对话
  "model": "deepseek-chat",    可选 默认 deepseek-chat
  "stream": false,             可选 是否流式输出
  "enable_thinking": false     可选 开启思考模式
}
成功响应 200
{
  "success": true,
  "conversation_id": 1,
  "message": {
    "role": "assistant",
    "content": "你好！我是..."
  },
  "tokens": {
    "prompt": 120,
    "completion": 45,
    "total": 165
  }
}
流式输出：设置 "stream": true 后，响应以 SSE 格式返回，Content-Type 为 text/event-stream，前端需使用 EventSource 或 fetch + ReadableStream 读取。
GET /api/chat/conversations 需认证

获取当前用户的对话列表，按更新时间倒序，支持分页。

Query 参数
?page=1&per_page=20
GET /api/chat/conversations/:id 需认证

获取指定对话的详细信息及全部消息记录。仅限操作自己的对话。

DELETE /api/chat/conversations/:id 需认证

删除指定对话及其所有消息。仅限操作自己的对话。

管理后台接口

所有 /api/admin/* 接口均需管理员权限，且不受接口开关限制（始终可用）。

GET /api/admin/dashboard

数据看板。返回总用户数、今日活跃、7 日对话趋势、模型分布、时段分布等统计。

GET /api/admin/users

用户列表，支持搜索、状态筛选、分页。

Query 参数
?page=1&per_page=20&search=关键词&status=active|disabled
PUT /api/admin/users/:id

更新用户信息。

{
  "role": "admin|user",
  "is_active": true|false,
  "daily_limit": 100  // null = 继承全局
}
POST /api/admin/users/:id/reset-password

重置用户密码为随机字符串并返回。

GET /api/admin/users/:id/conversations

查看指定用户的对话列表（管理员功能）。

GET /api/admin/conversations

全部对话概览（管理员功能）。

GET /api/admin/conversations/:id

查看任意对话详情（需开启 admin_can_view_content 配置）。

POST /api/admin/reset

系统重置 — 清空所有对话、消息、配置、用户（保留管理员），恢复出厂设置。

GET /api/admin/vendors

获取所有厂商及模型列表（含能力标签：text/streaming/thinking/vision/tools/file_upload/web_search）。

GET /api/admin/onboarding/status

获取新手引导状态。

POST /api/admin/onboarding/complete

完成新手引导，一站式配置管理员密码、AI 厂商、全局权限、注册开关等。

权限系统

Lumen 采用两级权限架构，灵活控制每个用户的能力边界。

两级权限架构

全局默认权限

定义所有新用户的初始能力。管理员在「权限设置」页面修改后，所有未设单独权限的用户立即生效。

用户单独权限

管理员可为特定用户设置自定义权限。设为「继承全局」则跟随全局默认值；设为「自定义」则覆盖全局。

权限项说明

权限项 类型 默认值 说明
max_daily_chats number 100 每日对话次数上限
allowed_models string[] deepseek-chat, deepseek-reasoner 允许使用的模型 ID 列表
max_tokens_per_request number 4096 单次请求最大 Token 数
rate_limit_per_minute number 10 每分钟请求频率限制
allow_export boolean false 是否允许导出对话
allow_file_upload boolean false 是否允许上传文件

用户每日限额

每个用户可设置独立的 daily_limit 字段。设为 null 表示使用全局 max_daily_chats；设为具体数值则覆盖全局。

GET /api/admin/permissions/global

获取全局默认权限配置。

PUT /api/admin/permissions/global

更新全局默认权限。变更后所有未设单独权限的用户立即生效。

GET /api/admin/permissions/user/:id

获取用户单独权限配置及有效权限合并结果。

PUT /api/admin/permissions/user/:id

更新用户单独权限。设置 use_custom: false 恢复继承全局。

接口开关

管理员可精细控制每个接口的启用/禁用状态。被禁用的接口返回 403。管理后台接口（/api/admin/*）始终可用，不受开关影响。

分组说明

认证 (auth)

注册、登录、登出、个人信息、改密

对话 (chat)

发送消息、对话列表、对话详情、删除对话

管理后台 (admin)

始终可用，不受开关控制

GET /api/admin/endpoints

获取所有接口开关列表（按分组）。

PUT /api/admin/endpoints/:id

切换单个接口的启用/禁用状态。

PUT /api/admin/endpoints/batch

批量切换接口开关。

{
  "endpoint_ids": [1, 2, 3],
  "enabled": true
}

自定义厂商

在管理后台「系统设置」中选择「自定义 (OpenAI 兼容)」厂商，填入任意兼容 OpenAI Chat Completions API 的 Base URL 和 API Key，即可接入第三方模型。支持 Ollama、vLLM、LocalAI 等本地部署方案。

系统配置

GET /api/admin/config

获取系统配置。API Key 等敏感字段返回掩码（如 sk-****xxxx）。

PUT /api/admin/config

更新系统配置。支持批量更新多个配置项。

GET /api/admin/config/api-key

获取真实 API Key（管理员专用），同时返回用量统计。

POST /api/admin/config/test-connection

测试 AI API 连接。发送一条简短测试消息并返回延迟与 Token 消耗。

{
  "vendor_id": "deepseek",
  "api_key": "sk-...",
  "base_url": "https://api.deepseek.com",
  "model": "deepseek-chat"
}
GET /api/admin/config/open-registration

获取开放注册开关状态。

PUT /api/admin/config/open-registration

切换开放注册开关。

部署

环境变量

变量名 默认值 说明
SECRET_KEY 自动生成 Flask 密钥，用于 JWT 签名
JWT_SECRET_KEY 自动生成 JWT 专用密钥
DATABASE_URL sqlite:///app.db 数据库连接串（支持 PostgreSQL）
DEPLOY_RUN_PORT 5000 服务监听端口

生产环境建议

  • 使用 PostgreSQL 替代 SQLite，设置 DATABASE_URL 环境变量
  • 设置强随机 SECRET_KEY 和 JWT_SECRET_KEY
  • 前端使用 Nginx 反向代理，开启 HTTPS
  • 定期备份 app.db（SQLite）或配置数据库自动备份
  • 生产环境建议使用 gunicorn 替代 Flask 开发服务器

使用 Gunicorn 启动

pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app

错误码

状态码 错误码 说明
400 missing_fields 缺少必填参数
401 unauthorized 未提供 Token 或 Token 无效/过期
403 forbidden 无权限（非管理员访问管理接口 / 接口已关闭）
403 endpoint_disabled 该接口已被管理员关闭
403 registration_closed 开放注册已关闭
403 daily_limit_reached 已达每日对话上限
404 not_found 资源不存在
409 conflict 用户名或邮箱已存在
500 internal_error 服务器内部错误
