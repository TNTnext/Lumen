快速开始

Quick Start

安装与启动

Installation & Launch

# 克隆项目 / Clone
git clone https://github.com/your-org/lumen.git
cd lumen

# 安装依赖 / Install dependencies
pip install -r requirements.txt

# 启动服务（默认端口 5000）/ Start server (default port 5000)
python app.py

第一个 API 请求

Your First API Request

1. 登录获取 Token / Login to get Token

curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

2. 发送对话 / Send a message

curl -X POST http://localhost:5000/api/chat/send \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_token>" \
  -d '{"message":"你好，请介绍一下你自己"}'

认证接口

Authentication APIs

所有认证接口均不需要 Token。除注册外，其余接口始终可用（不受接口开关影响）。

All auth endpoints do not require a Token. Except registration, all are always available (not affected by endpoint toggles).

POST /api/auth/register

注册新用户。需管理员开启开放注册开关，否则返回 403。

Register a new user. Requires admin to enable open registration, otherwise returns 403.

请求体 Request Body
{
  "username": "string",    必填 Required
  "password": "string",    必填 Required 最少 6 位 / min 6 chars
  "email": "string",       可选 Optional
  "daily_limit": 100       可选 Optional null=继承全局 / inherit global
}
成功响应 200 Success
{
  "success": true,
  "message": "注册成功",
  "token": "eyJ...",
  "user": { "id": 1, ... }
}
POST /api/auth/login

用户登录，返回 JWT Token（有效期 24 小时）。

User login, returns JWT Token (valid for 24 hours).

请求体 Request Body
{
  "username": "admin",
  "password": "admin123"
}
成功响应 200 Success
{
  "success": true,
  "token": "eyJ...",
  "user": { ... },
  "must_change_password": true
}
GET /api/auth/me 需认证 Auth Required

获取当前登录用户的详细信息。

Get current logged-in user details.

Headers
Authorization: Bearer <token>
POST /api/auth/change-password 需认证 Auth Required

修改当前用户密码。

Change current user's password.

请求体 Request Body
{
  "old_password": "current_password",
  "new_password": "new_strong_password"
}
POST /api/auth/logout 需认证 Auth Required

登出当前用户，使 Token 失效。

Logout current user, invalidate the Token.

对话接口

Chat APIs

核心 AI 对话功能，支持多轮上下文、流式输出、多模型切换。所有接口均需认证，受每日限额与权限控制。

Core AI chat features: multi-turn context, streaming, multi-model. All require auth, subject to daily limits & permissions.

POST /api/chat/send 需认证 Auth Required

发送消息并获取 AI 回复。自动加载对话历史构建多轮上下文。

Send a message and get AI reply. Automatically loads conversation history for multi-turn context.

请求体 Request Body
{
  "message": "你好",           必填 Required
  "conversation_id": 1,        可选 Optional 继续已有对话 / continue
  "model": "deepseek-chat",    可选 Optional 默认 / default deepseek-chat
  "stream": false,             可选 Optional 是否流式输出 / streaming
  "enable_thinking": false     可选 Optional 开启思考模式 / thinking mode
}
成功响应 200 Success
{
  "success": true,
  "conversation_id": 1,
  "message": {
    "role": "assistant",
    "content": "你好！我是..."
  },
  "tokens": {
    "prompt": 120,
    "completion": 45,
    "total": 165
  }
}
流式输出 / Streaming：设置 "stream": true 后，响应以 SSE 格式返回，Content-Type 为 text/event-stream，前端需使用 EventSource 或 fetch + ReadableStream 读取。
GET /api/chat/conversations 需认证 Auth Required

获取当前用户的对话列表，按更新时间倒序，支持分页。

Get current user's conversation list, sorted by update time desc, with pagination.

Query 参数 Query Params
?page=1&per_page=20
GET /api/chat/conversations/:id 需认证 Auth Required

获取指定对话的详细信息及全部消息记录。仅限操作自己的对话。

Get conversation details and all messages. Only accessible for own conversations.

DELETE /api/chat/conversations/:id 需认证 Auth Required

删除指定对话及其所有消息。仅限操作自己的对话。

Delete a conversation and all its messages. Only for own conversations.

管理后台接口

Admin APIs

所有 /api/admin/* 接口均需管理员权限，且不受接口开关限制（始终可用）。

All /api/admin/* endpoints require admin role and are always available (not affected by endpoint toggles).

GET /api/admin/dashboard

数据看板。返回总用户数、今日活跃、7 日对话趋势、模型分布、时段分布等统计。

Dashboard stats: total users, today's active, 7-day chat trends, model distribution, hourly distribution.

GET /api/admin/users

用户列表，支持搜索、状态筛选、分页。

User list with search, status filter, and pagination.

Query 参数 Query Params
?page=1&per_page=20&search=关键词&status=active|disabled
PUT /api/admin/users/:id

更新用户信息。

Update user information.

{
  "role": "admin|user",
  "is_active": true|false,
  "daily_limit": 100  // null = 继承全局 / inherit global
}
POST /api/admin/users/:id/reset-password

重置用户密码为随机字符串并返回。

Reset user password to a random string and return it.

GET /api/admin/users/:id/conversations

查看指定用户的对话列表（管理员功能）。

View a specific user's conversation list (admin feature).

GET /api/admin/conversations

全部对话概览（管理员功能）。

Overview of all conversations (admin feature).

GET /api/admin/conversations/:id

查看任意对话详情（需开启 admin_can_view_content 配置）。

View any conversation details (requires admin_can_view_content enabled).

POST /api/admin/reset

系统重置 — 清空所有对话、消息、配置、用户（保留管理员），恢复出厂设置。

System reset — clear all conversations, messages, configs, users (keep admin), restore factory settings.

GET /api/admin/vendors

获取所有厂商及模型列表（含能力标签：text/streaming/thinking/vision/tools/file_upload/web_search）。

Get all vendors and models with capability tags.

GET /api/admin/onboarding/status

获取新手引导状态。

Get onboarding wizard status.

POST /api/admin/onboarding/complete

完成新手引导，一站式配置管理员密码、AI 厂商、全局权限、注册开关等。

Complete onboarding: configure admin password, AI vendor, global permissions, registration toggle, etc.

权限系统

Permission System

Lumen 采用两级权限架构，灵活控制每个用户的能力边界。

Lumen uses a two-tier permission architecture for flexible per-user capability control.

两级权限架构

Two-Tier Permission Architecture

全局默认权限

定义所有新用户的初始能力。管理员在「权限设置」页面修改后，所有未设单独权限的用户立即生效。

Defines initial capabilities for all new users. Changes take effect immediately for users without custom permissions.

用户单独权限

管理员可为特定用户设置自定义权限。设为「继承全局」则跟随全局默认值；设为「自定义」则覆盖全局。

Admins can set custom permissions for specific users. "Inherit" follows global defaults; "Custom" overrides them.

权限项说明

Permission Items

权限项 Item 类型 Type 默认值 Default 说明 Description
max_daily_chats number 100 每日对话次数上限 / Daily chat limit
allowed_models string[] deepseek-chat, deepseek-reasoner 允许使用的模型 ID 列表 / Allowed model IDs
max_tokens_per_request number 4096 单次请求最大 Token 数 / Max tokens per request
rate_limit_per_minute number 10 每分钟请求频率限制 / Rate limit per minute
allow_export boolean false 是否允许导出对话 / Allow conversation export
allow_file_upload boolean false 是否允许上传文件 / Allow file upload

用户每日限额

User Daily Limit

每个用户可设置独立的 daily_limit 字段。设为 null 表示使用全局 max_daily_chats；设为具体数值则覆盖全局。

Each user can have an independent daily_limit. Set to null to inherit global max_daily_chats; set a specific number to override.

GET /api/admin/permissions/global

获取全局默认权限配置。

Get global default permission configuration.

PUT /api/admin/permissions/global

更新全局默认权限。变更后所有未设单独权限的用户立即生效。

Update global default permissions. Takes effect immediately for all users without custom permissions.

GET /api/admin/permissions/user/:id

获取用户单独权限配置及有效权限合并结果。

Get user's custom permission config and effective merged permissions.

PUT /api/admin/permissions/user/:id

更新用户单独权限。设置 use_custom: false 恢复继承全局。

Update user's custom permissions. Set use_custom: false to revert to global inheritance.

接口开关

Endpoint Toggles

管理员可精细控制每个接口的启用/禁用状态。被禁用的接口返回 403。管理后台接口（/api/admin/*）始终可用，不受开关影响。

Admins can finely control each endpoint's enabled/disabled state. Disabled endpoints return 403. Admin endpoints (/api/admin/*) are always available.

分组说明

Groups

认证 (auth)

注册、登录、登出、个人信息、改密

Register, Login, Logout, Profile, Change Password

对话 (chat)

发送消息、对话列表、对话详情、删除对话

Send, List, Detail, Delete conversations

管理后台 (admin)

始终可用，不受开关控制

Always available, not affected by toggles

GET /api/admin/endpoints

获取所有接口开关列表（按分组）。

Get all endpoint toggles (grouped).

PUT /api/admin/endpoints/:id

切换单个接口的启用/禁用状态。

Toggle a single endpoint's enabled/disabled state.

PUT /api/admin/endpoints/batch

批量切换接口开关。

Batch toggle endpoint states.

{
  "endpoint_ids": [1, 2, 3],
  "enabled": true
}

自定义厂商

Custom Vendors

在管理后台「系统设置」中选择「自定义 (OpenAI 兼容)」厂商，填入任意兼容 OpenAI Chat Completions API 的 Base URL 和 API Key，即可接入第三方模型。支持 Ollama、vLLM、LocalAI 等本地部署方案。

In the admin panel "System Settings", select "Custom (OpenAI Compatible)" vendor, enter any OpenAI Chat Completions API-compatible Base URL and API Key to connect third-party models. Supports Ollama, vLLM, LocalAI, and other local deployment solutions.

System Configuration

Get system config. Sensitive fields like API Key return masked values (e.g. sk-****xxxx).

Update system config. Supports batch updating multiple config items.

Deployment

环境变量

Environment Variables

变量名 Variable 默认值 Default 说明 Description

生产环境建议

Production Recommendations

  • 使用 PostgreSQL 替代 SQLite，设置 DATABASE_URL 环境变量 / Use PostgreSQL instead of SQLite
  • 设置强随机 SECRET_KEY 和 JWT_SECRET_KEY / Set strong random secrets
  • 前端使用 Nginx 反向代理，开启 HTTPS / Use Nginx reverse proxy with HTTPS
  • 定期备份 app.db（SQLite）或配置数据库自动备份 / Regular database backups
  • 生产环境建议使用 gunicorn 替代 Flask 开发服务器 / Use gunicorn instead of Flask dev server

使用 Gunicorn 启动

Launch with Gunicorn

pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
