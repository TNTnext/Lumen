API 接口文档

Lumen 提供完整的 RESTful API，支持用户认证、AI 对话、管理后台等功能。所有接口（除登录/注册外）均需携带 JWT Token 进行身份认证。

认证接口

POST /api/auth/register

用户注册（需开放注册开关开启）

请求体
{
  "username": "string (必填)",
  "password": "string (必填, 最少6位)",
  "email": "string (可选)",
  "daily_limit": "number (可选, 每日对话限额)"
}
响应
{
  "success": true,
  "message": "注册成功",
  "token": "jwt_token_string",
  "user": { "id": 1, "username": "...", ... }
}
POST /api/auth/login

用户登录，返回 JWT Token

请求体
{
  "username": "string (必填)",
  "password": "string (必填)"
}
响应
{
  "success": true,
  "token": "jwt_token_string",
  "user": { ... },
  "must_change_password": true
}
GET /api/auth/me

获取当前用户信息（需认证）

Headers
Authorization: Bearer <token>
POST /api/auth/change-password

修改当前用户密码（需认证）

请求体
{
  "old_password": "string",
  "new_password": "string"
}

对话接口

POST /api/chat/send

发送消息并获取 AI 回复（需认证，受每日限额限制）

请求体
{
  "message": "string (必填)",
  "conversation_id": "number (可选, 继续已有对话)",
  "model": "string (可选, 默认 deepseek-chat)"
}
响应
{
  "success": true,
  "conversation_id": 1,
  "message": { "role": "assistant", "content": "..." },
  "tokens": { "prompt": 100, "completion": 50, "total": 150 }
}
GET /api/chat/conversations

获取当前用户的对话列表（分页，需认证）

Query 参数
page=1&per_page=20
GET /api/chat/conversations/:id

获取对话详情及消息记录（需认证，仅限自己的对话）

DELETE /api/chat/conversations/:id

删除对话（需认证，仅限自己的对话）

管理后台接口（需管理员权限）

GET /api/admin/dashboard

数据看板统计（总用户数、今日活跃、对话趋势等）

GET /api/admin/users

用户列表（支持搜索、状态筛选、分页）

Query 参数
page=1&per_page=20&search=关键词&status=active|disabled
PUT /api/admin/users/:id

更新用户信息（角色、状态、每日限额）

请求体
{
  "role": "admin|user",
  "is_active": true|false,
  "daily_limit": 100  // null 表示使用全局默认
}
POST /api/admin/users/:id/reset-password

重置用户密码

GET /api/admin/permissions/global

获取全局默认权限配置

PUT /api/admin/permissions/global

更新全局默认权限

GET /api/admin/permissions/user/:id

获取用户单独权限配置（含有效权限合并结果）

PUT /api/admin/permissions/user/:id

更新用户单独权限

GET /api/admin/config

获取系统配置（API Key 为掩码）

GET /api/admin/config/api-key

获取真实 API Key（管理员专用，含使用统计）

PUT /api/admin/config

更新系统配置

POST /api/admin/config/test-connection

测试 DeepSeek API 连接

GET /api/admin/config/open-registration

获取注册开关状态

PUT /api/admin/config/open-registration

切换注册开关

用户管理接口（需管理员权限）

GET /api/auth/users

用户列表（支持搜索、状态筛选、分页）

PUT /api/auth/users/:id

更新用户（角色、状态、邮箱、每日限额）

请求体
{
  "role": "admin|user",
  "is_active": true|false,
  "email": "string",
  "daily_limit": 100  // null 表示使用全局默认
}
POST /api/auth/users/:id/reset-password

重置用户密码

DELETE /api/auth/users/:id

删除用户（不能删除唯一管理员）

权限系统说明

两级权限架构

  • 全局默认权限：定义所有新用户的初始能力，包括每日对话次数、可用模型、Token 限制等
  • 用户单独权限：管理员可为特定用户设置自定义权限，覆盖全局默认值

权限项说明

权限项 说明 默认值
max_daily_chats 每日对话次数上限 100
allowed_models 允许使用的模型列表 deepseek-chat, deepseek-reasoner
max_tokens_per_request 单次请求最大 Token 数 4096
rate_limit_per_minute 每分钟请求频率限制 10
allow_export 是否允许导出对话 false
allow_file_upload 是否允许上传文件 false

用户每日限额

每个用户可设置独立的 daily_limit 字段，控制该用户每日可发送的对话次数。设置为 null 表示使用全局默认权限中的 max_daily_chats 值。

默认管理员账号

用户名 admin
密码 admin123

首次登录后系统会提示修改密码，建议立即修改为强密码。
